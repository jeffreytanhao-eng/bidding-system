# 产品需求文档 (PRD)

## 1. 项目概述
- **项目名称**: 招标管理系统
- **版本**: v1.0
- **创建日期**: 2026-04-20
- **目标用户**: 企业采购部门、招标负责人、评审专家、供应商

## 2. 核心目标
为企业提供一套完整的招标管理解决方案，实现从供应商管理、标书发布、应标评审到中标公示的全流程数字化管理，提高招标效率和透明度。

## 4. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 页面加载时间 < 3秒；支持100个供应商同时应标 |
| 安全 | 用户认证、数据加密、操作日志记录、敏感数据脱敏 |
| 兼容性 | 支持Chrome、Firefox、Edge最新版本；钉钉小程序支持 |
| 可用性 | 系统可用性 > 99.5% |
| 可扩展性 | 支持横向扩展，未来可支持多租户 |

## 5. 约束条件

- 技术约束：需与钉钉开放平台集成、需配置企业邮箱服务
- 时间约束：MVP版本需在4周内完成
- 资源约束：小型开发团队（2-3人）

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 标书 | 招标方发布的招标文件，包含招标要求、条件等信息 |
| 应标 | 供应商响应招标，提交投标文件的过程 |
| 商务评审 | 对供应商报价、资质、业绩等商务因素的评审 |
| 技术评审 | 对供应商技术方案、团队能力等技术因素的评审 |
| 中标 | 供应商在招标中获胜，获得项目合同 |
| 钉钉组织架构 | 企业在钉钉中的部门和人员结构 |

## 7. 附录

### 7.1 业务流程图

```
招标流程：
供应商入库 → 标书创建 → 选择供应商 → 发布标书（邮件）
    ↓
供应商应标（商务+技术文件）
    ↓
指派评审人员（钉钉）→ 商务评审 + 技术评审（并行）
    ↓
评分汇总 → 生成推荐名单 → 审核确认
    ↓
结果归档 + 更新供应商记录
```
